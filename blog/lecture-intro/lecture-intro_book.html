        변수(40p)
        
            - 데이터 저장과 참조를 위해 할당된 메모리 공간
        
                - 변수선언
        
                    - 예시
                    
                        int num;
        
                        - 해석
                        
                            int : 데이터 타입이 정수인 값을 저장할 메모리 공간을 할당하겠습니다.
                            
                            num : 메모리 공간에 접근할 때는 num을 쓰겠습니다.
        
                    - 변수 num에 값(10)을 저장
        
                        num = 10;
        
                    - 변수 선언과 동시에 저장
        
                        int num2 = 20;
        
                    - 변수 여러개 동시 선언(값 저장도 가능)
        
                        double num1, num2;
        
                    - 변수값 대상 연산
        
                        int num3 = num + num2;
        
                    - 변수 이름짓기
        
                        - 숫자로 시작 안됨
        
                        - ‘_’와 ‘$’ 이외 특수문자 안됨
        
                        - 키워드(자바의 문법을 구성하는 단어들) 안됨
        
                    - 변수 데이터 타입
        
                        - boolean	T / F		1byte		True o false
        
                        - char		문자		2bytes		유니코드 문자
        
                        - byte		integer		1byte		2^8 범위(-128~127)
        
                        - short		integer		2bytes		2^16 범위(±3만2천정도)
        
                        - int		integer		4bytes		2^32 범위(±21억정도)
        
                        - long		integer		8bytes		2^64 범위(±많음)
        
                        - float		float		4bytes		
        
                        - double	float		8bytes	
        
        
        정수 표현방식 이해하기(46p)
            - 2진수 8진수 16진수 기본 이해 가정
        
            - 1바이트 정수표현
        
                0  0  0  0  1  0  1  1 ==> 정수 +11
        
                MSB(Most Signigicant Bit) : 첫 번째(0양수 1음수, 부호를 알려주는 비트)
        
            - 음수 표현법
        
                00000101	==> 정수 +5 표현
                
                11111010	==> 1의 보수(비트 반전)
        
                11111011	==> 2의 보수(+1) ==> 정수 –5
        
            - 비트 연산 중에 올림수는 버린다
        
        
        실수 표현방식 이해하기(49p)
        
            - 1과 2사이의 실수는 몇 개인가? ==> 무한개
        
            - 정밀도를 낮추고 근사치를 표현하자.
        
            - 실수 표현 수식
        
                ±(1.m)*2^(e-127)
        
                위 수식에 반영하는 비트구성
        
                  1 00000 1011101000 ==>		(sign)(e)(m)
        
            - m이나 e 의 변동에 따른 차이가 큼
        
                num1 = 1.0000001;
        
                num2 = 2.0000001;
        
                num1 + num2 ==> 3.00000019999997 //오차발생
        
            - 참고
        
                IEEE : 전기전자공학 최대 기술조직
        
                    “IEEE754” 실수표현방식 정의
        
                    32비트 표현인 single precision, 64비트 표현인 double precision
        
        
        자바 자료형
        
            - 각 타입별 참조 방식이 다르기 때문에 자료형을 결정해 주어야한다.
        
                - 정수 자료형 : byte, short, int, long
        
                - 자바는 정수형 연산을 int형으로 진행한다. ==> long 제외
        
                    short num1 = 1;
        
                    short num2 = 2;
        
                    short result = num1 + num2; // 연산을 int형으로 진행했기 때문에 강제 형변환 필요
        
                - 실수 자료형 : float(6자리 정밀도), double(15자리 정밀도)
        
                - 문자 자료형 : char
        
                    Character Set : Unicode
        
                    char ch1 = ‘헐’;		==> 저장할 때 실제 저장되는 것은 해당 문자 유니코드
        
                    char ch2 = 54736;	==> ‘헐’의 유니코드(컴파일러가 할 일을 대신한 것)
        
                    char ch3 = 0xD5D0;	==> 유니코드 16진수 표현(해당 16진수 앞에 0x를 붙인다)
        
                - 논리 자료형 : boolean
        
                    true o false
        
        
        상수(64p)
        
            상수(constant) : 값이 변하지 않는 수
        
                - 자바의 일반적인 상수
        
                    - 변수 선언시 앞에 final 선언 추가하면 상수가 됨
        
                      변경이 불가, 딱 한번만 할당할 수 있음
        
                    - 관례상 상수의 이름은 대문자
                    
                      두단어 이상이면 언더바
        
                - 정수형 상수 표현법
        
                    - 예시
        
                        10진수 : int num = 10;
        
                        2진수 : byte seven = 0b111;	==> 앞에 0b o 0B 입력
        
                        8진수 : int num = 011;		==> 앞에 0 입력
        
                        16진수 : int num = 0x11;	==> 앞에 0x o 0X 입력
        
                    - 자바는 byte, short 정수 표현법 제공 x
        
                    - long은 숫자 끝에 ‘L’ 입력
        
                        long num = 314L;
        
                    - 단위가 큰 수는 언더바 이용
        
                        int num = 100_000_000;		==> 편한 곳에 넣어도됨
        
                - 실수형 상수 표현법
        
                    - 기본은 double, 명시하고 싶다면 끝에 d o D 입력
        
                    - float는 끝에 f o F 입력
        
                - 참고
        
                    - e o E
        
                        3.4e3 = 3400.0
        
                        3.4E3f 표현법과 같이 사용도 가능
        
                    - 0 생략
                    
                        0.5	=> .5
        
                        7.0f	=> 7.f
        
        
        Escape sequence
        
            - \b	백스페이스
        
            - \t	tab
        
            - \\	\
        
            - \’	’
        
            - \“	“
        
            - \n	new line
        
            - \r	carriage return, cursor를 왼쪽 끝으로 이동시켜 overwrite
        
            - \u	유니코드 입력방법, 유니코드 값 앞에 사용 ex) 유로 단위 기호 입력 방법 ‘\u20AC’
        
        
        형변환(72p)
        
            - 타입별 표현방법이 달라서 연산 시에도 다름, 자료형 변환 과정이 필요함
        
            - 자동적 형변환 과정 : 프로그래머가 명시한 형변환 x, 필요한 상황에 자동으로
        
                - 규칙
        
                    - 자료형의 크기 상관없이 정수보다 실수 우선
        
                    - 자료형의 크기가 큰 방향으로
        
                        byte < short < int < long < float < double
                                char <
        
            - 명시적 형변환 과정
        
                - 예시1
        
                      double pi = 3.14;
        
                      int i = (int)pi;		==> pi 값을 int 형으로 명시적 형변환
        
                - 예시2 : 연산은 int로 바꿔서 연산하므로 주의
        
                      short num1 = 1;
        
                      short num2 = 2;
                      
                      short num3 = num1 + num2;		==> 오류
                      
                      short num3 = (short)(num1 + num2);
